  Description:
    Abstract: These data were originally developed and designed for Rhode Island Enhanced 9-1-1 purposes only. These data represents every known fire hydrant in the state of Rhode Island. These data include, both addressed and unaddressed structures including occupied and unoccupied structures.
    Purpose: These data were designed to provide information about fire hydrant locations in Rhode Island in support of the RI E-911 Uniform Emergency Telephone System. These data are provided to RIGIS for redistribution as a public service by RI E-911.
    Supplemental_Information:
      Portions of this data set were collected as early as 2001. Inaccuracies do exist in these data and are therefore under constant revision. Any discrepancies, inaccuracies or inconsistencies recognized in these data should be reported to the pertinent municipality who should alert RI E-911
      Users are also encouraged to email ri911gis@akassociates911.com with any suggested updates for this actively maintained dataset.

Data_Quality_Information:
  Attribute_Accuracy:
    Attribute_Accuracy_Report: These data is subject to constant revision and updating. An updated list of landline telephone numbers is downloaded monthly. The street address of each telephone number is matched up with an address existing in the GIS database using x9GIS Ali-Auditor. Telephone numbers that fail to match are then classified as missing addresses and GIS personnel take action to locate the specific address and location. Municipal Tax and GIS data is utilized when available to locate missing addresses. New and missing site locations were collected and inaccuracies were rectified using GPS receivers in the field. These site locations are generally expected to be within 5  10 meters of its true location, but variations in GPS data collection quality can create variable results. This dataset includes all known new sites built between 2001 and the date this dataset was shared with RIGIS.
  Logical_Consistency_Report: This dataset is considered to be topologically clean.

  Lineage:
    Process_Step:
      Process_Description: This layer was originally developed by manually digitizing sites from 1:5000 scale spring 1997 aerial photography. Every known address in the state was then field was then field verified over the course of seven years. The field verification process involved vehicle mounted GPS units in concert with X9collecter GIS software. A site picture of every known address was also taken at the time and linked to the site. This work was completed by MicroDATA GIS on behalf of RI E-911. On a daily basis, verification data from the field was imported into the master data set, and checked by a GIS technician. As field-verification was completed in each municipality, an extensive set of automated and manual quality control audits were completed to verify attribute accuracy, to reconcile GIS data with telephone company MSAG and ALI data, as well as data from local officials, and to verify route connectivity. After a period of municipal review, any final changes or corrections provided by local officials were completed, and GIS Sites data was published for use by RI-911.

    Attribute:
      Attribute_Label: Measure
      Attribute_Definition: This is the distance, in miles, from the site (or access point) to beginning of the road the site is on. It is automatically created when a site is added. Or, it is updated by Running Addressing Utilities. It is used for calculating the site address based on the addressing rules and by microDATA's Assessment Report Module software
    Attribute:
      Attribute_Label: ParcelNum
      Attribute_Definition: This is the parcel number that the site is located in. Direct table editing is required to update this field. This is used for nothing except the x9GIS user's reference. This data exists in only select communities and is not maintained by E-911.
    Attribute:
      Attribute_Label: ZN
      Attribute_Definition: This is the name of the MSAG/ALI community that this site spatially falls within.
    Attribute:
      Attribute_Label: HouseNumbe
      Attribute_Definition: Primary street address number assigned to a site.  It should generally match what is on the addressing authority's records.
    Attribute:
      Attribute_Label: PrimaryAdd
      Attribute_Definition: This is the primary "official" or "legal" address field, it should generally match what is on the street sign OR what the addressing authority's records indicate. This is kept in sync with the PrimaryName in the GeoNames table matching the GeoNameID.
    Attribute:
      Attribute_Label: ALIAddress
      Attribute_Definition: This is the entire address of the site, made up of the HouseNumber and the ALIName fields. This is updated by any changes made to the GeoNameID or HouseNumber fields. This address information reflects the address associated with a landline telephone number to pinpoint 9-1-1 callers.
    Attribute:
      Attribute_Label: PrimaryNam
      Attribute_Definition: This is the primary "official" or "legal" street name field, it should generally match what is on the street sign OR what the addressing authority's records indicate. This is kept in sync with the PrimaryName in the GeoNames table matching the GeoNameID.
    Attribute:
      Attribute_Label: ALIName
      Attribute_Definition: This is the Street Name field with any variation of spelling found in the telephone company MSAG or ALI database of the geoname the house is addressed with.
    Attribute:
      Attribute_Label: PD
      Attribute_Definition: This is the Prefix Direction parsed from ALIName Field in the GeoNames table.
    Attribute:
      Attribute_Label: PT
      Attribute_Definition: This is the Prefix Type parsed from ALIName Field in the GeoNames table. It is updated by making changes to the GeoNames table's PD, PT, SN, ST, SD. It is used for matching any incoming ALI-String when using 911 mapping software.
    Attribute:
      Attribute_Label: SN
      Attribute_Definition: This is the Street Name parsed from ALIName Field in the GeoNames table. It is updated by making changes to the GeoNames table's PD, PT, SN, ST, SD.
    Attribute:
      Attribute_Label: ST
      Attribute_Definition: This is the Street Type parsed from ALIName Field in the GeoNames table. It is updated by making changes to the GeoNames table's PD, PT, SN, ST, SD.
    Attribute:
      Attribute_Label: SD
      Attribute_Definition: This is the Suffix Direction parsed from ALIName Field in the GeoNames table. It is updated by making changes to the GeoNames table's PD, PT, SN, ST, SD.
